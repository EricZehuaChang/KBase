# CHANGELOG

## v1.0.0（2026-07-15）—— 首个产品版本

**质量基线**：后端 pytest 449 passed（external/pg 按设计跳过）；前端 vitest 107 passed + vue-tsc 构建通过；核心能力均经浏览器端到端实测。

### RAG 核心链路

- 摄取三条解析路（上传时可选）：`auto`（markitdown + 扫描件自动路由 GLM-OCR）、`ocr` 表格增强（文本层 PDF 强制结构化解析，跨页断表合并）、`vlm` 满血视觉深度识别（识别结果人工校验确认后才向量化，防幻觉入库）
- 表格全套：原子分块（永不切碎）+ 大表按行组切分重复表头 + 行线性化（"列=值"进向量与 BM25）+ 跨页断表合并
- 父子分块 + 双路召回（向量 + BM25 关键词）+ RRF 融合 + 重排（可降级）+ 小到大上下文组装
- 检索策略三层可编排：全局默认 → KB 级用法开关 → 请求级试跑覆盖
- 引用溯源：文本层 PDF 页码定位 + 原文件内联预览跳页；回答附命中页文档插图（多模态一期，图片不进 prompt 零幻觉）；命中即溯源（表格/图片定位到对应对象，不做 bbox 高亮——产品决策）

### 模型接入（全部页面配置化）

- LLM Provider：页面增删改查 + API Key 页面维护（脱敏显示）+ 连通性测试 + 各厂商模型目录拉取（7 天缓存周更）；预置智谱/通义/DeepSeek/Kimi/硅基流动/OpenAI 及企业自有 OpenAI 兼容平台
- 向量模型：KB 级绑定（建库选择）+ 密钥页面维护（DB 覆盖 > 环境变量）+ admin 换绑触发全库重建（存量文本重嵌入，零 OCR 费用）
- OCR：GLM-OCR（智谱 layout_parsing）；VLM 深度识别 provider 可配

### 企业能力

- 认证鉴权：本地账号（viewer/editor/admin 角色矩阵）+ API Key + 企业 SSO（标准 OIDC 授权码流，默认关闭）
- 库级权限 ACL："不配公开、一配收紧"（授权用户 + 建库人 + admin）
- 多库联合问答：会话绑定多库，散射-聚合跨库检索，引用带来源库
- OpenAI 兼容 API：每个知识库暴露为 `/v1/chat/completions` 的"模型"，生态客户端零改造接入
- 反馈闭环：回答点赞点踩 + 差评清单（带问题原文）入运营看板
- 运营看板：问答量/拒答率趋势 + 无答案问题清单 + 赞踩统计
- 评测回归：评测集 + 一键回归（hit@k / MRR，只评检索零 LLM 费用）+ 历史对比
- URL 连接器：网页地址直接导入（内网 wiki/门户）
- Chunk 运营：分块启停（索引摘除可恢复）+ 文本编辑重嵌入

### 部署与运维

- 双形态：lite（SQLite+Chroma+进程内 bge-m3，POC/小团队）/ standard（PostgreSQL+Qdrant+TEI，生产，实测可支撑十万级文档）
- 备份恢复：`scripts/backup.py`（SQLite 在线快照不停机备份、恢复可回滚）+ 运维手册（备份剧本/升级五步/监控接入）
- 可观测：`/healthz` + `/metrics`（Prometheus）
- 批量导入：`kbase/bulk_import.py` CLI（断点续传/失败重跑，十万级灌库）
- 许可证：Ed25519 验签，试用/到期前 30 天/过期三级横幅提醒（不锁功能）
- 演示数据一键装载（幂等）

### 前端

- 双 SPA：使用端（问答/会话/多库联查/反馈/插图灯箱）+ 管理端（Vben 风骨架：分组导航/激活指示条/面包屑，统一 PageHeader）
