### Task H1: TEI Embedder 与 Reranker 适配器
kbase/plugins/embedders/tei.py（POST {endpoint}/embed {"inputs":[...]} → 向量组;dimension 惰性探测一次）与 kbase/plugins/rerankers/tei.py（POST /rerank {"query","texts"} → [{index,score}] 按原序返回分数）。注册名 "tei"。错误面：不可达抛 RuntimeError（reranker 由既有降级路径接住;embedder 失败=摄取/查询失败,如实抛）。TDD：httpx MockTransport 假 TEI（正常/乱序 rerank 响应重排/超时/非 2xx）。config：embedder/reranker 各加 endpoint 字段。
Commit `feat: TEI embedder/reranker HTTP 适配器`

### Task H2: Qdrant VectorStore 适配器
kbase/plugins/vectorstores/qdrant_store.py 实现 VectorStore 契约（upsert/search/delete/delete_collection/get_vectors;cosine;collection 惰性建,维度首次 upsert 时定）。单测用 `QdrantClient(":memory:")`——把 tests/test_chroma_store.py 的用例提炼成共享契约测试函数,两适配器同跑（重构该文件为参数化,Chroma 与 Qdrant 一致过）。config vectorstore 加 endpoint/api_key 可选字段。
Commit `feat: Qdrant 向量库适配器（契约测试参数化共享）`

### Task H3: PG 元数据与 PG 关键词索引
config 加 `db: {url: str = "sqlite:///./data/kbase.sqlite"}`（create_app 改用 cfg.db.url,默认不变——注意现状是 f-string 拼 data_dir,保持 sqlite 默认语义兼容:url 中支持 {data_dir} 占位替换）。migrations 方言感知：SQLite 走现有路径;PG 跳过 FTS5/ALTER 语法差异处理（PG 的 IF NOT EXISTS 语义,信息 schema 查列）。kbase/index/keyword_pg.py：chunks_kw 表（chunk_id/kb_id/doc_id/tsv tsvector）+GIN;index/search/delete_doc/delete_kb 与 FTS5 版同契约（jieba 预分词沿用 _tokenize）;工厂函数 make_keyword_index(sf, dialect) 自动选择。PG 集成测试标记 `@pytest.mark.pg`（默认跳过,`KBASE_TEST_PG_URL` 设置时跑）——纯 SQL 形状/工厂选择逻辑用单测覆盖。
Commit `feat: PG 元数据支持与 PG 关键词索引（方言感知迁移）`

### Task H4: Docker 镜像与双 compose
Dockerfile（python:3.11-slim,分层缓存:先依赖后代码;entrypoint.sh 等待 db/qdrant 可达[standard]→uvicorn --host 0.0.0.0）;.dockerignore;docker-compose.lite.yml（app+卷:data、hf-cache）;docker-compose.standard.yml（app、postgres:16-alpine、qdrant/qdrant、ghcr.io/huggingface/text-embeddings-inference 两实例[embed bge-m3 / rerank bge-reranker-v2-m3],healthcheck+depends_on,GPU 段注释与 CPU 变体注释,国内镜像源注释）;config/kbase.standard.yaml 样例（指向 compose 服务名）。本地无 docker 则语法校验（`docker compose config` 不可用时用 yaml 解析）+ 留 H5 真实构建。

### Task H6: 100 并发压测与调优
