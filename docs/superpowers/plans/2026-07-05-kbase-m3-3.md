### Task F2: JobRunner 与大纲生成

**Files:** Create `kbase/jobs/runner.py`、`kbase/jobs/proposal.py`（本任务仅 outline 部分）；Test `tests/test_job_runner.py`

- runner.py：`run_job(sf, job_id, steps: list[tuple[str, Callable]])`——顺序执行，每步前 update progress `{steps:[{name,status:running|done|failed,detail}]}`，步骤抛异常→该步 failed+记 detail，**继续后续步骤**；全部结束：任一步 failed → status=done_with_errors（无产出步骤 failed 则 failed），否则 done。同步函数（BackgroundTasks 承载）。
- proposal.py `generate_outline(retriever, llm, kb_id, topic, requirements) -> list[{title, brief}]`：检索 topic top-5 作背景 → LLM complete（prompt 要求输出 JSON 数组）→ **鲁棒解析**：剥 ```json 围栏、找首个 [ 到末个 ]、json.loads 失败→ValueError 带原文摘录。async（调用方 asyncio.run 或在 async 端点直接 await——outline 走同步端点，用 await）。
- 测试：runner 三态（全 done/中间失败继续/全失败）+ progress 逐步写入断言；outline 解析四例（干净 JSON/围栏包裹/前后杂文/坏 JSON 抛错）。FakeLLM。
- Commit `feat: JobRunner 顺序编排与大纲生成（鲁棒 JSON 解析）`

### Task F3: 逐节生成与汇整（引用全局重编号）

**Files:** Modify `kbase/jobs/proposal.py`；Test `tests/test_proposal_flow.py`

- `generate_section(retriever, llm, kb_id, topic, section) -> {title, text, citations}`：检索 `f"{topic} {section['title']} {section['brief']}"` top-5 → usable 门（复用 Generator.usable_blocks 语义——直接实例化 Generator 用其 _build_messages/citations，complete 非流式）→ 无可用块时 text=占位说明（"知识库中无相关依据，本节未生成"）citations=[]。
- `assemble(topic, sections_out) -> str`：Markdown：`# {topic}` + 各节 `## title` + 正文（节内 [n] 重映射：全局引用表按 (doc_name, heading_path) 去重编号，正文 [n] 正则替换为全局号）+ 末尾 `## 引用文献` 列表 `[g] doc_name › heading_path`。纯函数可测。
- `build_proposal_steps(...) -> steps`：为 runner 生成步骤列表（每节一步 + 汇整一步 + 写产物一步——artifact md 落 data/jobs/{id}/artifact.md）。
- 测试：重编号纯函数（两节引用同一文档合并为同号、不同出处递增、正文替换正确）；无依据节占位；全流程（FakeLLM+fake 检索）产物 md 结构断言。
- Commit `feat: 逐节生成与汇整（引用全局重编号+无依据占位）`

### Task F4: 汇编流与 docx 导出

**Files:** Create `kbase/jobs/digest.py`、`kbase/jobs/export_docx.py`；Modify `pyproject.toml`（python-docx）；Test `tests/test_digest_export.py`

- digest.py：`build_digest_steps(sf, retriever?, llm, kb_id, doc_ids|None, job_id, files_dir)`——每 ready 文档一步：读 content.md 前 6000 字 → LLM 摘要段（200 字内，prompt 固定）；最后总览步（各摘要合并请求一段总览）+ 汇整 md（`# {kb名}文档汇编` + 总览 + 每文档 `## filename` 摘要）+ 产物写盘。
- export_docx.py：`markdown_to_docx(md: str, out_path)`——支持 #/##/### → Heading1-3、空行分段、`- ` 列表、`**bold**` run；其余原样段落。python-docx。
- 测试：digest 全流程（Fake 组件）产物结构；docx 转换后用 python-docx 读回断言标题层级与段落数；粗体 run。
- Commit `feat: 定期汇编流与 Markdown→docx 导出`
