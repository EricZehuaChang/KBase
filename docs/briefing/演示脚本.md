# KBase 现场演示脚本

- 演示日期：2026-07-08（周三）
- 使用对象：现场操作演示的同事
- 目标：在不确定现场网络环境的前提下，稳妥、可控地完成三大场景 + 安全机制的演示

> 本脚本假设**不启动任何新服务**进行彩排验证——所有"启动服务"步骤请在演示前于本机独立完成并确认可用，本文档只描述演示当天的操作顺序与话术。

## 一、演示前检查清单

请在出发前（建议提前一天 + 当天早上各过一遍）逐项确认：

### 1. 本地 lite 栈可正常启动

```powershell
# 加载 .env 中的密钥
Get-Content -Encoding utf8 .env | ForEach-Object {
    if ($_ -match '^([A-Z_0-9]+)=(.+)$') {
        Set-Item -Path "env:$($Matches[1])" -Value $Matches[2]
    }
}
.venv\Scripts\uvicorn --factory kbase.api.main:create_app --port 8100
```

- [ ] 浏览器打开 `http://localhost:8100` 能看到登录页
- [ ] `/healthz` 返回全部插件状态正常（无红色/异常标记）
- [ ] 首次启动如需下载 bge-m3/reranker 模型，**务必提前完成**，避免现场等待 60~120 秒加载

### 2. DashScope（大模型网关）连通性测试 —— 注意 IPv6 坑

已知问题：部分网络环境下，DashScope 域名解析优先走 IPv6，而现场网络的 IPv6 出口可能不通或极慢，表现为"问答一直转圈、最终超时"。请提前用以下方式验证并强制走 IPv4：

```powershell
# 方式一：直接测试连通性（观察是否秒回）
Measure-Command { Invoke-WebRequest -Uri "https://dashscope.aliyuncs.com/compatible-mode/v1/models" -Headers @{Authorization="Bearer $env:DASHSCOPE_API_KEY"} -TimeoutSec 8 }

# 方式二：若怀疑走了 IPv6 导致缓慢，强制解析到 IPv4 地址再测试
$ipv4 = (Resolve-DnsName dashscope.aliyuncs.com -Type A).IPAddress | Select-Object -First 1
Write-Host "IPv4 地址：$ipv4"
Invoke-WebRequest -Uri "https://dashscope.aliyuncs.com/compatible-mode/v1/models" -Headers @{Host="dashscope.aliyuncs.com"; Authorization="Bearer $env:DASHSCOPE_API_KEY"} -TimeoutSec 8
```

- [ ] 上述请求能在 3 秒内返回（而非卡到超时）
- [ ] 若确认是 IPv6 路由问题，最简单的现场应急是**关闭系统网络适配器的 IPv6**（控制面板 → 网络适配器 → 属性 → 取消勾选 IPv6），或在 hosts 文件里把 `dashscope.aliyuncs.com` 固定指向上面解析出的 IPv4 地址
- [ ] 演示前用真实问答走一遍完整链路（问答页问一个问题，确认能流式出字）

### 3. 演示知识库：6 份文档就绪

- [ ] 演示知识库（兵团政策演示库，`3f59dc2f-e480-407a-b370-812d3162bf97`）应有 6 份 `ready` 文档：北屯市186团小镇发展规划.docx、北屯市综合物流枢纽基地创建方案.pdf、兵团本级机关事业单位工作人员差旅费管理办法-新兵办发〔2014〕76号.pdf、各师市协助用餐用车标准.png（扫描件，用于展示 OCR）、最新住宿标准.xlsx、公务卡报销费用注意事项.docx
- [ ] 全部文档状态为 `ready`（知识库管理页确认，无 `pending_ocr`/`failed` 残留）
- [ ] 扫描件文档（各师市协助用餐用车标准.png）确认 OCR 已完成、置信度正常显示

### 4. 账号就绪

- [ ] `admin` 账号：密码已知（查启动日志或 `.env` 里的 `KBASE_ADMIN_PASSWORD`），可正常登录
- [ ] 一个 `viewer` 角色账号已提前创建（设置页 → 用户管理），用于演示权限差异
- [ ] 两个账号都验证过能成功登录，避免现场输错密码卡壳

### 5. 提前跑好的方案生成成品任务

- [ ] 方案生成实测约需 439 秒（7 节），**现场不能干等**，务必提前发起一次生成任务并等到 `done`
- [ ] 确认该成品任务的 Markdown 产物与 docx 导出都能正常打开
- [ ] 记下该任务在"生成"页任务历史列表中的位置，演示时能快速找到

### 6. 投屏分辨率与显示

- [ ] 投屏前确认分辨率（建议 1920×1080，与 slides.html 的 16:9 设计匹配）
- [ ] 幻灯片 `slides.html` 用浏览器全屏打开（F11），确认键盘 ← → 翻页正常
- [ ] 深色背景幻灯片在现场灯光/投影下的可读性提前看一眼（深色主题在强光环境下对比度会打折扣）
- [ ] 准备好第二个浏览器窗口/标签页用于切换到实际系统演示，避免频繁切换应用造成卡顿感

### 7. 断网应急预案素材准备（见下文风险预案表）

- [ ] 提前录制一份完整演示流程的屏幕录像（含财务问答多轮追问、检索分析、生成产物展示），作为断网时的备用播放素材
- [ ] 确认手机热点可用，且热点下 DashScope 连通性也提前测过一次

## 二、逐步演示流程

建议总时长 15~20 分钟，可根据现场时间灵活裁剪（标 ⚠️ 的环节是关键风险点）。

### 步骤 1：登录

操作：用 `admin` 账号登录系统。

> **讲解要点**：KBase 默认开启认证，全部接口都需要有效身份，这是私有化交付的基本门槛。

### 步骤 2：财务报销多轮问答（核心场景三）

操作：

1. 在问答页提问："北京出差，部级干部住宿费标准是多少？"，等待流式回答完成（答案应含"800"元，出自《差旅费管理办法》附件2）；
2. 点击回答中的引用角标 `[n]`，打开引用抽屉，展示命中片段与"查看全文"跳转；
3. 追问："那司局级呢？"，展示系统依然准确理解上下文并给出正确答案"500"元（而非因为字面检索不到而拒答）。

> **讲解要点**：追问背后是查询改写机制——系统把"那司局级呢"这类指代性追问，在检索前自动改写成完整问题，但用户看到的问题和历史记录不受影响。

### 步骤 3：检索分析页——四路对比（讲混合检索价值）

操作：在检索分析页输入"新兵办发〔2014〕76号文件是关于什么内容的？"这类包含精确文号的查询，展示稠密路（向量）/关键词路/RRF 融合/重排前后对比四栏。

> **讲解要点**：政策文件里大量文号、机构名等精确表述，纯语义检索容易漏召回；关键词路负责把这些精确词捞回来，两路融合再重排，兼顾语义理解与精确匹配。

### 步骤 4：知识库页——展示已完成 OCR 的扫描件

操作：打开知识库管理页，找到`各师市协助用餐用车标准.png`这份扫描件文档，展示其状态为 `ready`、OCR 置信度正常；回到问答页提问"第一师出差人员在内部食堂用餐，早中晚餐分别收费多少？"验证这份"看起来是图片"的资料确实能被检索命中并给出正确数值。

> **讲解要点**：政策文件里大量是扫描件，系统能自动识别并转换为可检索文本，扫描 PNG 样本实测 13 秒完成识别，置信度达到 1.0；识别质量不足的文档会标黄提示人工核对，不会静默出错。

### 步骤 5：生成向导（⚠️ 关键风险点：不现场等待）

操作：

1. 在生成页发起一次新的方案生成任务（填主题、要求，展示大纲生成与编辑）——**发起后立即切换**，不要在现场等待生成完成；
2. 切到提前准备好的**已完成**任务，展示完整产物：正文分节、引用编号、"引用文献"附录；
3. 下载并打开 docx 导出，展示格式转换效果。

> **讲解要点**：方案生成是"大纲同步生成、正文逐节异步生成"，7 节完整方案实测约 439 秒，全节成功；某节若确实没有依据，系统会明确写"本节未生成"而不是编造内容。

### 步骤 6：设置页——角色/审计/许可证

操作：展示用户管理（三级角色）、API Key 管理、审计日志列表、许可证状态。

> **讲解要点**：这是私有化交付的合规基础——谁能做什么事、做过什么事，全部可查可控。

### 步骤 7：切换 viewer 账号，展示权限差异

操作：登出 admin，用 viewer 账号登录，尝试访问知识库管理/设置页，展示入口被隐藏或访问被拒绝（403）。

> **讲解要点**：权限是路由级强制校验的，不是前端"藏起来"就完事——即使直接改 URL 访问，后端也会拒绝。

## 三、风险预案表

| 风险场景 | 应对预案 |
|---|---|
| 现场完全断网 | lite 栈本身全部本地运行（检索、向量化、界面均不依赖外网），但 **LLM 生成仍需要外网访问 DashScope**。预案：① 立即切换手机热点（提前测过可用）；② 若热点也不可用，播放提前录制好的完整演示屏幕录像，讲解话术照常进行 |
| DashScope 连接超时/卡顿（IPv6 路由问题） | 按检查清单第 2 项执行：临时关闭 IPv6 或改 hosts 强制走 IPv4 地址；若来不及现场排查，直接切手机热点 |
| 现场提问跑偏、问到语料库外的内容 | 回到准备好的问题清单（见下），礼貌引导："这个问题超出了今天演示知识库的范围，但系统的处理方式是……"，顺势展示拒答机制而非勉强回答 |
| 生成任务现场卡住或报错 | 不排查，直接切换到提前准备好的成品任务继续演示，事后再复盘原因 |
| 投影分辨率与设计不匹配、幻灯片显示异常 | slides.html 是纯静态单文件，可在任意现代浏览器全屏打开，无需网络；提前用现场实际投影设备走一遍即可发现问题并调整浏览器缩放比例（Ctrl +/-） |
| viewer 账号密码忘记/登录失败 | 提前打印/记录好两个账号的用户名密码，随身携带一份纸质备份 |

## 四、现场问答准备：8 个问题清单

以下问题全部基于演示知识库**实测内容**设计（2026-07-06 用 `eval/run_eval.py --tiers` 逐题验证过检索命中，见 `eval/report-tiers.md`），前 6 个应当**必命中**并给出准确带引用的回答，后 2 个是**故意问域外内容**，用于展示拒答机制而非临场发挥。

### 必命中（6 个，覆盖三大场景、混合检索与 OCR）

1. **北京出差，部级干部住宿费标准是多少？**（验证基础检索命中，答案应含"800"元；出自《差旅费管理办法》附件2；场景三多轮问答的第一问，为下一问追问做铺垫）
2. **那司局级呢？**（紧接上一问追问，验证 QueryRewrite 指代改写，答案应准确给出"500"元且不拒答——系统把"那司局级呢"在检索前自动改写成完整问题）
3. **员工出差在外，交通费、住宿费和伙食补助费分别应该怎么报销，是凭据报销还是包干使用？**（自然语言长问题，验证系统对口语化、多子问题叠加的表述依然能正确检索并综合作答；答案应区分"凭据报销"与"包干使用"）
4. **新兵办发〔2014〕76号文件是关于什么内容的？**（**文号精确检索**，讲解要点：精确文号是关键词路召回的关键词，纯向量常因语义太"稀"而丢失这类专有表述，混合检索+关键词路能稳定捞回）
5. **第一师出差人员在内部食堂用餐，早中晚餐分别收费多少？**（**OCR 扫描件验证**：答案来自`各师市协助用餐用车标准.png`——一张扫描表格图片，讲解要点：系统对扫描件先做 OCR 识别转文本再入库检索，这道题证明"看起来是图片"的资料一样能被语义/关键词检索命中，而不是只能人工翻阅）
6. **五万元以下的零星购买支出应该怎么结算？**（验证关键词路对专有表述"公务卡"的精确匹配，出自《公务卡报销费用注意事项》）

### 展示拒答机制（2 个，故意问域外问题）

7. **住房补贴的申领条件是什么？**（M1 演示曾用过这道题，但当前演示知识库**已不包含"住房补贴办法"这份文档**——这恰好是一次天然的拒答演示：系统应明确回答"未找到依据"而非编造，讲解时可以坦诚提及"此前版本的知识库里有这份文档，现在库内容换了，系统诚实地说不知道，而不是硬凑答案"）
8. **竞品友商的报销系统多少钱一套？**（与知识库主题完全无关的问题，用于展示系统不会为了"显得有用"而胡乱作答）

> 若临场提问超出以上 8 题范围，优先引导回到清单内问题；确需回答域外提问时，坦诚说明"这是根据当前演示知识库内容的判断，实际部署时知识库范围由客户决定"。
