背景与需求

三个真实场景，一个共同难点

① 政策方案生成

给定主题 → 大纲 → 逐节检索生成 → 全文引用重编号 → docx 导出

② 政策摘编 / 定期汇编

多文档定期汇总，逐文档摘要 + 总览，供领导快速通览

③ 财务报销多轮问答

连续追问（"那司局级呢？"）需要理解上下文指代，不能拒答

共同难点：答案必须有据可查、专有名词不能漏召回、数据不能出域、AI不能编造。

总体方案

分层可插拔架构

接入层Web 管理与问答界面 / REST API / MCP Server
应用层逐轮问答(RAG) / 定期汇编 / 方案生成（内置编排）
内核层摄取 → 分块 → 索引 → 混合检索(+重排) → 生成
插件层Embedder / VectorStore / LLMProvider / OCR / Chunker（均可替换）
存储层向量库 / 元数据库 / 原始文件+Markdown 双存

同一套内核代码，配置切换即可从单机演示（lite）平滑升级到集群生产（standard）。

场景三 · 财务报销多轮问答

追问式提问，系统听得懂上下文

"出差住宿费标准是多少？" → "那司局级呢？"

QueryRewrite 查询改写

检索前用 LLM 把指代性追问改写为语义完整、可独立检索的问题；仅影响检索这一步，回答与会话记录仍用用户原文。

三态触发，避免误改写

启发式判断：问题较短 / 含指代词 / 与历史几乎无重叠才触发；完整问题不受影响。改写失败静默回退原问题，不阻塞主流程。

核心可信机制

引用溯源与防幻觉门

全链路引用溯源

每句答案的引用角标可点击，弹出引用抽屉查看命中片段、相关度分数，并可跳转原文全文定位。

拒答门

证据相关度不足时明确回答"未找到依据"，不编造；拒答计入运营指标而非错误。

冲突证据保留

检索到冲突信息时不做静默取舍，保留证据交模型辨析并提示用户注意。

混合检索

向量语义 + 关键词双路 + RRF 融合 + 重排，文号/人名/专有名词精确词不丢。

扫描件处理

OCR 自动识别，纳入同一套检索管线

自动路由

文档摄取时自动探测是否有文本层；扫描件/图片路由至 MonkeyOCR（GPU 服务）转换为 Markdown 后进入统一分块/索引流程；服务不可用时优雅降级为"待 OCR"，不阻塞批次。
